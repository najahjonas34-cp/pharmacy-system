# Contributing to Pharmacy Management System

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch (`git checkout -b feature/amazing-feature`)
4. Make your changes
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Code Style

- Use ES6+ syntax
- Follow ESLint configuration
- Use meaningful variable names
- Add comments for complex logic

## Testing

Before submitting a PR:
- Run `npm test` in both backend and frontend
- Ensure all tests pass
- Add tests for new features

## Issues

Before creating an issue, check if it already exists. When creating an issue:
- Use a clear, descriptive title
- Provide detailed description
- Include steps to reproduce (for bugs)
- Include expected behavior and actual behavior

## Pull Requests

- Keep PRs focused on a single feature or fix
- Write clear commit messages
- Update documentation as needed
- Reference related issues
